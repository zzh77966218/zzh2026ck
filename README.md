# RemoteLink - 远程控制解决方案

一个类似 RustDesk 的远程控制解决方案，包含 Windows 服务器端和 Android 客户端。

## 🎯 核心特点

- **ID直连** - 无需手动输入IP，扫码/输入ID即可连接
- **自动发现** - 客户端自动注册到服务器
- **中继转发** - 不同网络也能通过服务器转发控制
- **简洁易用** - 类似RustDesk的现代化界面

## 🚀 快速开始

### 1️⃣ 启动服务器 (Windows)

```powershell
cd server
pip install -r requirements.txt
python server.py
```

**打包为EXE:**
```cmd
cd server
build.bat
```

### 2️⃣ 构建APK (Android)

```bash
cd android_client

# 安装依赖
pip install -r requirements.txt

# 构建 (Linux/WSL)
buildozer android debug
```

### 3️⃣ 安装APK

```bash
adb install bin/remotecontrol-*-debug.apk
```

## 📱 使用方法 (RustDesk风格)

### 场景一：扫码连接 (推荐)

```
┌─────────────┐         📱 扫描          ┌─────────────┐
│   服务端    │  ←─── 二维码 ───→  │   客户端    │
│   显示ID    │                      │   打开相机   │
│   + 二维码  │                      │   扫码连接   │
└─────────────┘                      └─────────────┘
```

1. 服务端启动后，界面显示 **6位ID** 和 **二维码**
2. 客户端点击「**扫码连接**」
3. 用手机相机扫描服务端二维码
4. 自动连接，开始控制

### 场景二：ID输入连接

```
┌─────────────┐                      ┌─────────────┐
│   服务端    │  ←─── ID: 123456 ──→  │   客户端    │
│   ID: 123456│                      │   输入ID    │
│             │                      │   点击连接   │
└─────────────┘                      └─────────────┘
```

1. 服务端界面显示6位ID (如 `847293`)
2. 客户端在主界面看到「**在线设备列表**」
3. 点击列表中的设备，或手动输入ID
4. 连接成功，开始控制

### 场景三：自动连接

- 客户端启动后**自动搜索**本地网络
- 若找到服务器，自动获取分配的本机ID
- 在线设备列表**实时更新**

## 🔧 技术架构

### 通信流程

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌──────────┐         ┌──────────┐         ┌──────────┐   │
│   │ 客户端A  │────────→│ 注册服务  │←────────│ 客户端B  │   │
│   │ (Viewer) │         │ :21115   │         │ (Host)   │   │
│   └──────────┘         └──────────┘         └──────────┘   │
│         │                    │                    │         │
│         │                    │                    │         │
│         ↓                    │                    ↓         │
│   ┌──────────┐               │             ┌──────────┐    │
│   │ 中继服务 │←───────────────┼────────────→│  主机    │    │
│   │ :21116  │               │             │ 控制处理 │    │
│   └──────────┘               │             └──────────┘    │
│         │                    │                    │         │
│         └────────────────────┴────────────────────┘         │
│                         数据转发                            │
└─────────────────────────────────────────────────────────────┘
```

### 端口说明

| 端口 | 用途 |
|------|------|
| 21115 | 注册服务 - 客户端注册、ID分配、连接协商 |
| 21116 | 中继服务 - 远程控制数据转发 |

### ID机制

- 6位数字ID，由服务器自动分配
- 格式：`100000-999999`
- 每个客户端有唯一ID
